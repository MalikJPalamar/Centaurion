# BuilderBee Delivery

## Purpose

Manage client delivery for BuilderBee, including GHL (GoHighLevel) setup, client onboarding, automation building, and ongoing client support.

## Systems

- **GHL (GoHighLevel):** Primary platform for client CRM, funnels, automations, and communications
- **Client Portal:** Onboarding checklists, project tracking, and deliverable handoffs
- **Automation Library:** Reusable workflow templates for common client needs

## Procedure

### 1. Client Onboarding
- Run onboarding audit: what does the client currently have? What needs to be built?
- Set up GHL sub-account with standard configuration
- Import relevant automation templates from the library
- Schedule kickoff call and share onboarding checklist

### 2. Automation Build
- Map client's business process to GHL workflow
- Build automations: lead capture → nurture → booking → follow-up
- Test all trigger points and notification paths
- Document custom configurations for future reference

### 3. Ongoing Support
- Monitor automation performance (conversion rates, error rates)
- Handle client change requests and scope adjustments
- Identify upsell opportunities based on usage patterns

## Example

For example, a typical client onboarding: audit reveals they have a WordPress site but no CRM — set up GHL with contact import, build a lead capture funnel, create a 5-email nurture sequence, and configure appointment booking with calendar sync.

## Routing

Most delivery tasks are medium-stakes, reversible → auto-execute. Exceptions:
- Scope changes that affect pricing → route to Malik
- Client escalations or complaints → route to Malik
- New automation patterns not in the library → document and route for review
