# BuilderBee Client Onboarding Workflow

## Purpose

Standardized workflow for onboarding new BuilderBee clients from signed contract to live GHL setup.

## Steps

### 1. Pre-Onboarding Audit
- Review client's current tech stack (website, CRM, email, scheduling)
- Identify what can be migrated vs. what needs to be built fresh
- Document client's top 3 business goals for the engagement

### 2. GHL Setup
- Create GHL sub-account with standard BuilderBee template
- Configure custom domain and branding
- Import contact list (if existing CRM data available)
- Set up pipeline stages matching client's sales process

### 3. Automation Configuration
- Build lead capture automation (form/funnel → CRM → notification)
- Create nurture email sequence (5-7 emails, customized to client's voice)
- Set up appointment booking with calendar integration
- Configure missed-call text-back and after-hours auto-reply

### 4. Testing & QA
- Test every automation trigger end-to-end
- Verify email deliverability (SPF, DKIM, DMARC)
- Confirm calendar sync and booking notifications
- Run a test lead through the full pipeline

### 5. Client Handoff
- Record walkthrough video of their GHL dashboard
- Share login credentials and documentation
- Schedule 30-day check-in call
- Set up monitoring alerts for automation failures

## Timeline

Target: 5 business days from contract signing to live setup.

## Routing

Standard onboarding is routine — auto-execute. Escalate to Malik if:
- Client requests custom integrations outside GHL
- Scope exceeds the contracted package
- Technical blockers require third-party vendor coordination
