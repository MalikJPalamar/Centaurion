# The Precision Ratio

```
Precision = Predictive Order / Thermodynamic Cost = -ε / C
```

## Definition

**Precision** — in Active Inference, the confidence-weight assigned to predictions. The Precision Ratio measures how sharply a system models its environment relative to the cost of maintaining those models. This is Centaurion's first principle — every design decision traces back to this ratio.

> Future evolution: The **Coherence Equation** will extend the Precision Ratio to measure alignment between human and AI substrates — how well the centaur's two halves share a unified model. Precision is individual acuity; Coherence is composite alignment.

## Components

### Predictive Order (-ε)
The negative of prediction error. When prediction error is low, predictive order is high. A system with high predictive order:
- Anticipates what the user needs before being asked
- Classifies tasks correctly on the first pass (routing accuracy)
- Surfaces relevant context without being prompted
- Makes recommendations that align with human judgment

**Measured by:** Routing accuracy, recommendation acceptance rate, Malik's outcome ratings (1-5), wiki coverage of encountered topics.

### Thermodynamic Cost (C)
The total resources consumed to maintain the system:
- **Time:** How long does Malik spend directing, reviewing, correcting?
- **Money:** API costs, VPS hosting, tool subscriptions
- **Cognitive load:** How much mental effort does context-switching require?
- **Energy:** Compute cycles, token consumption

**Measured by:** Time-to-completion trends, monthly API spend, context-switch frequency (how often Malik must re-explain).

## Optimization Strategies

### Increase Predictive Order
1. **Memory compounds:** Every interaction feeds Supermemory, wikis, Graphiti. The system gets better with use.
2. **Routing improves:** The Routing Gate's thresholds adjust based on outcome data. Fewer misroutes over time.
3. **Identity deepens:** TELOS files accumulate context. New agents start with full knowledge of Malik's preferences, ventures, and history.

### Decrease Thermodynamic Cost
1. **Automate the routine:** Anything that's been done correctly 3+ times should be a workflow, not a manual task.
2. **Phone-first:** No desktop required = no context-switching to a different device = lower cognitive cost.
3. **Async by default:** Agents run overnight, Malik reviews in the morning. No synchronous waiting.
4. **Free tiers first:** Supermemory free tier, GitHub Actions free tier, self-hosted VPS. Scale cost only when value is proven.

## The Precision Gradient

The system is on a precision gradient when:
- Malik's ratings trend upward over weeks
- Time-per-review trends downward
- Wiki coverage increases without manual effort
- Routing accuracy improves (fewer misclassifications)
- Context-switch cost decreases (system remembers across ventures)

If any of these trends reverse, it's a signal to diagnose and correct. The daily health check monitors these metrics.
